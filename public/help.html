Job Ranger Help

Private job monitoring, explained clearly.

Job Ranger helps you monitor selected career pages, review results locally, and tune the desktop runtime without turning your search into a subscription or a cloud workflow.

Best use case Track a focused list of companies instead of trawling broad job boards.
Core workflow Add source, run scrape, review jobs, tighten filters, repeat.
Storage model Companies, jobs, filters, and settings are stored locally on your machine.

Getting Started

  1. Open the app and wait for the dashboard to load.
  2. Move to Companies and choose Add source.
  3. Enter a company name, a careers URL, and a polling frequency.
  4. Run the first scrape and review the results in Jobs.
Implemented desktop workflow

Adding Sources

Job Ranger works best when you add explicit company career pages that you actually care about. It classifies source types rather than assuming every site is equally automatable.

  • Supported: adapter-backed paths such as Greenhouse, Lever, SmartRecruiters, and Ashby.
  • Detected: sources such as Workday, iCIMS, BambooHR, Taleo, Oracle Careers, and generic HTML pages.
  • Browser-required: portals that need a browser-backed extraction path.
  • Unsupported: pages without a reliable extraction route.
If a source is marked unsupported, the app is being honest. That status is preferable to a false success result.

Reviewing Jobs

The Jobs page is the review queue for your locally stored results. Use it to inspect title, location, salary text, and matched filter counts, then mark items seen as you process them.

Using Filters

Filters help reduce noise after the first few scrapes. You can combine title terms, keyword terms, salary minimums, and location rules to make the next pass more useful.

  • Start narrow rather than encoding every preference up front.
  • Use title include rules for role families.
  • Use keyword exclude rules to remove noisy categories.
  • Adjust based on the match counts you see after a run.

Settings

The settings screen exposes two kinds of controls.

  • Runtime behavior: user agent, timeout, concurrency, retries, and cooldown values.
  • Desktop facts: database path, SQLite path, and supported adapters.

These are not cosmetic toggles. They directly shape how assertive the local worker is when it scrapes supported sources.

Notifications And Tray

Job Ranger can be configured to notify on new jobs, matched jobs, and minimize to the tray when the main window is closed. Use the settings page to control this behavior.

The application menu also provides an in-app Help entry and a shortcut for opening the local data folder.

Troubleshooting

  • If no jobs appear, confirm that the source URL is a real careers page and the source is active.
  • If a source is unsupported, treat that as a truthful classification rather than a transient bug.
  • If notifications do not appear, verify both the app toggles and your OS notification permissions.
  • If closing the window exits the app, enable minimize-to-tray in settings.
  • If the desktop backend cannot start in a development environment, confirm that sqlite3 is available or set SQLITE3_PATH.
